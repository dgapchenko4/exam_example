## Вариант 4. Стоматологическая клиника

### Описание проекта

Небольшой сайт стоматологической клиники с несколькими врачами, перечнем услуг и записями на приём. 
Основные существующие модели и шаблоны в проекте должны меняться по минимуму. 
Упор делается на создание новых доменных моделей и аккуратное удаление нерелевантных.

### Задача

Реализуйте функциональность для работы стоматологической клиники, опираясь на уже существующую структуру проекта.

В рамках задания студент должен:

- **Добавить новые модели**
  - Врач (стоматолог) с базовой информацией и специализацией.
  - Услуга (например, лечение кариеса, чистка, имплантация) с ценой и длительностью.
  - Запись на приём (пациент, врач, услуга, дата/время, статус).
  - При необходимости — дополнительные сущности (например, кабинет), если это упрощает работу с расписанием.

- **Минимально доработать существующие модели и шаблоны**
  - При необходимости добавить связи от уже существующих сущностей к новым моделям (ForeignKey, ManyToMany и т.п.).
  - Изменять существующие модели и ключевые шаблоны только в тех местах, где это действительно нужно для работы сценария.

- **Удалить лишние модели и связи**
  - Найти в проекте модели и поля, которые не подходят под сценарий стоматологической клиники (например, специфичные для интернет-магазина товаров).
  - Удалить их из кода и убедиться, что не осталось «битых» импортов и обращений к этим моделям/полям.

- **Обеспечить отображение данных**
  - Настроить минимальные изменения в шаблонах, чтобы отобразить:
    - список врачей и их специализаций;
    - список стоматологических услуг с ценами;
    - список или детальную страницу записей на приём (например, для администратора/регистратора).
  - По возможности переиспользовать существующие базовые шаблоны и общие блоки, ограничившись точечными изменениями.

### Требования

- Основные базовые модели и ключевые шаблоны менять по минимальному объёму.
- Новые доменные сущности должны быть реализованы как отдельные модели.
- Ненужные для данного сценария модели и поля должны быть удалены.
- Проект после изменений должен успешно запускаться и корректно мигрировать базу данных.
